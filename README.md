# Application Security Auditing Lab

This repository is a hands-on application security auditing project focused on building small applications, identifying their vulnerabilities, exploiting and remediating those vulnerabilities, and documenting the whole process. 

The purpose of this project is to build small, intentionally imperfect applications, break them, and then fix them.

Each project follows a consistent workflow:

1. Design an implement a small application
2. Audit the code and runtime behavior
3. Identify vulnerabilities, undefined behaivor, and crash conditions
4. Exploit or demonstrate the issue
5. Apply secure fixes and document the mitigation

# Why this Exists

Security is best learned by doing, and I felt that by trying to build my own applications and test them, I could learn why vulnerabilities exist, and how they can be discovered through audits and how to fix them.

# Contents

- Small, self-contained applications
- Documented security findings with technical explanations
- Proof-of-concept exploit or crash demonstrations

# Goals

- Build proficiency in secure coding and defensive programming
- Identify, exploit, and remediate application-level vulnerabilities
- Strengthen code review and security auditing skills
- Apply security principles to real, working applications
